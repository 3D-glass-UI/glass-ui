About this project

Objectives for this project

  • Recreate as closely as possible the Vision Pro spatial UI styling.

    As depicted in the keynote since I don’t have access to the real thing

  • Only use CSS that is broadly supported in 2023.

    Definition of “broadly supported” for this project: works/looks good on the latest (desktop & mobile) versions of Firefox, Safari, and Chrome.

  • Create UI that is visible over almost any backdrop without the use of light/dark mode.

    This is useful not just for AR but also for video-based applications, 3D software, and diegetic/spatial UI in video games.

Q & A

What is the point of this project?

This project is an educational exploration of modern CSS techniques, not a commercial plug-and-play UI solution.
I set out to mimic the Apple Vision Pro spatial UI to see if it was possible and to challenge myself to play with all the shiny new CSS properties. I did not do it for monetary gain or to encourage people to copy other’s work and pass it off as their own. Not only because it would potentially be ethically/legally dubious but also because it would make for an extremely dull digital landscape.

That is why I designed the 3d glass UI library with ease of customization at its core. I hope you will have fun being creative while making it your own!

But why the Vision Pro's UI?

Because it displays an impressive level of craftsmanship and creativity that I seldom encounter in this category. Generally, UI is a means to an end. Select a primary and secondary color, add a font, and decide how round the corners should be. Done! Now move on to the real work. Fair enough, UI is there to enable the user to access the functionality of a thing, and not a thing in and of itself. Be that as it may, I can’t help but swoon in the presence of great beauty and innovation.

Why not create a unique glass look instead?

I did, several actually. But to be used in other projects. You don't them here because it’s tricky to objectively judge the success of a set of techniques via a unique creation. That’s because there is nothing concrete to compare the results to. Any artifact in the end product could be a mistake, a limitation, or a design decision.

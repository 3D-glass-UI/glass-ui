Have yourself a little CPU cookout

Here's the deal…any text or content you display over a glass background will be a lot more legible if said glass has a very generous blur radius. Unfortunately, blurring doesn't come cheap. And we are just getting warmed up! Next we we add soft drop shadows, and inset box-shadows stacked four layers deep, not to mention filters, blend modes, and the noise texture! All updating with every scroll.

So, it might be wise to exercise the type of restraint that this website lacks…and refrain from modeling your product’s entire interface on the Crystal Palace. Frosted glass

Main element

.frosted-glass {
  position: relative;

  box-shadow: 
   inset 2px 2px 0.4px -2.4px rgba(250, 254, 255, 0.7),
   inset 0.8px 0.8px 0.4px rgba(250, 254, 255, 0.1),
   inset 3px 3px 0.1px -3px rgba(250, 254, 255, 0.1),
   inset -1.4px -2px 1.5px -3.25px rgba(250, 254, 255, 0.7),
   inset -2px -2px 0.1px -2.4px rgba(250, 254, 255, 0.1);

  /* Add your own noise texture here! */
  background-image: url(/assets/images/noise-texture-500x500.png);
  background-size: 100px;
  background-repeat: repeat;
}

The main element...

  • Must

    be set to position absolute or relative

  • Must

    Not

    have a z-index

  • Must

    Not

    use any property that creates a new stacking context

Note

If you apply vibrant glass to an element and it looks wrong or seethrough, then a new stacking has been context is created by another class or in a papent element. You can either fix the stacking context or just use frosted glass instead.

::after

.frosted-glass::after {
  position: absolute;
  content: "";
  inset: 0;
  border-radius: inherit;
  z-index: -2;
  background-color: rgba(250, 254, 255, 0.12);
  mix-blend-mode: multiply;
}

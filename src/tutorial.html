Step one

Have yourself a little CPU cookout

Here's the deal…any text or content you display over a glass background will be a lot more legible if said glass has a very generous blur radius. Also, it just looks better. Unfortunately, like so many things that are easy on the eyes, blurring doesn't come cheap.

You see when you blur something, it's like you're asking each pixel to call up everybody in the neighborhood to ask, "What are you wearing today?" Then, after gathering all the gossip, it decides what to wear itself. Want a 20px blur? That's 20 chatty calls in every direction from every output pixel. And we are just getting warmed up! Next we we add soft drop shadows, and inset box-shadows stacked four layers deep, not to mention filters, blend modes, and the noise texture! All updating with every scroll.

So, it might be wise to exercise the type of restraint that this site lacks…and refrain from modeling your product’s entire interface on the Crystal Palace. Or, you know what? Do it. Use all the glass. Every possible piece of glass, all at once, everywhere, and let's just watch the world burn in all its high-resolution glory. Please do share screenshots!

   inset -2px -2px 0.1px -2.4px rgba(250, 254, 255, 0.1);

  /* Add your own noise texture here! */

  background-size: 100px;
  background-repeat: repeat;
}

Note

If you apply vibrant glass to an element and it looks wrong or seethrough, then a new stacking has been context is created by another class or in a papent element. You can either fix the stacking context or just use frosted glass instead.

::after

.frosted-glass::after {
  position: absolute;
  content: "";
  inset: 0;
  border-radius: inherit;
  z-index: -2;
  background-color: rgba(250, 254, 255, 0.12);
  mix-blend-mode: multiply;
}

Beveled glass tutorial

Vibrant glass uses the mix-blend-mode property to add dynamic depth of color to the glass material.

View vibrant glass tutorial

Hide vibrant glass tutorial

Abandon all hope, ye who's wondered thus far.' Be warned that this technique is quite finicky to implement and will stop working if applied to an element that has its own stacking context. The complexity arises from the interplay of multiple CSS properties, that each automatically create a new stacking context. Furthermore, mix-blend-mode needs to be set up such that it has access to the page’s backdrop to bend with. Finally, it is important to keep text above the glass so that its legibility is not affected.
